# Deferred adapters and sensitive data

The following are explicit post-v1 programs, not hidden v1 obligations:

- Atlas metadata projection and redacted export acknowledgements.
- Detached/on-cluster HPC worker bundles and reconciliation.
- R and Julia environment/execution contracts.
- Windows release gating.
- Real-time collaboration and hosted invitations.
- Cross-project publication repositories.
- Always-multi-agent and user-composed agent orchestration.
- Private/air-gapped security profiles and institutional deployment approval.
- Sensitive-data support and any compliance certification.
- Automatic learned-skill promotion or RSI.

Deferred adapters must use the same domain ports, stable IDs, ledger events, approval semantics, run envelope, audit dimensions and eval baselines. No adapter can make Atlas authoritative, bypass the controlled runner, or weaken administrator policy.

Sensitive-data readiness requires a separately approved threat model, fail-closed provider/network allowlists, OS process/network isolation, secret brokerage, encrypted operational state, egress canary tests, retention policy, incident process, and institution/data-owner authorization for each data class.
