# Work packet template

## Objective

One concrete implementation outcome.

## User-visible result

What a researcher can do when this packet lands.

## Normative inputs

- Requirement IDs:
- ADRs:
- Producer interfaces:

## Preconditions

Dependencies that must already be merged.

## Ownership

Production and test paths this packet may change. Parallel packets must not overlap.

## Invariants

Rules that must remain true across every transport and failure.

## Implementation

Small ordered steps. Consumers may not invent missing producer interfaces.

## Failure and recovery

Expected errors, preserved state, retries, cancellation and rollback.

## Tests

Real implementation tests and fixtures; avoid duplicated logic or mocks.

## Non-goals

Explicitly excluded work.

## Handoff

Interfaces, generated artifacts, commands run, results, limitations and follow-up packets.
